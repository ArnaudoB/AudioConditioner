\block{\MakeUppercase{Application I: Image to Ambient Music}}{
    One direct and powerful application of the \textbf{Audio\-Conditioner} framework is generating ambient music and atmospheres based solely on visual inputs. By passing an image into our pipeline:
    \vspace{0.5cm}
    
    \begin{itemize}
        \item The \textbf{BLIP Model} automatically captions the image's scene, extracting emotional elements, colors, objects, and implicit narrative.
        \item This scene description drives the \textbf{Audio\-Conditioner} pipeline (through the \textbf{CLAP Descriptor} and \textbf{Stable Audio}), outputting high-fidelity audio that perfectly encapsulates the mood of the image.
        \item The audio candidates are re-ranked using the \textbf{CLAP Cosine Similarity Filter} to ensure the highest alignment with the initial visual.
    \end{itemize}
    \vspace{0.5cm}
    This enables seamless soundtrack generation for static artwork, game environments, or visual media.
}

\block{\MakeUppercase{Application II: Text Reading with Adaptive Music}}{
    The \textbf{ReadMusic} module merges Speech Synthesis with adaptive background music scoring, turning textual narratives into fully immersive audiobooks.
    
    \begin{center}
    \begin{tikzpicture}[
        node distance=1.5cm and 1.5cm,
        box/.style={rectangle, draw=black, thick, fill=blue!10, text width=6cm, align=center, rounded corners, minimum height=1.5cm},
        arrow/.style={->, thick, >=stealth}
    ]
        \node[box, fill=green!10] (input) {Input Narrative Text};
        \node[box] (chunking) [below=1.5cm of input] {Text Chunking \\ (Max 80 words/chunk)};

        \node[box, fill=orange!10] (ac) [below right=2cm and 0.5cm of chunking] {\textbf{Audio\-Conditioner} \\ Generate Music \\ Match Duration};
        
        \node[box] (mixer) [below=10cm of chunking] {Length Matching \\ \& Dual Mixer ($Gains$)};
        \node[box, fill=yellow!10] (fade) [below=1.5cm of mixer] {Cross-fade Appending};
        \node[box, fill=green!10] (output) [below=1.5cm of fade] {Merged Audio Output};

        \draw[arrow] (input) -- (chunking);
        \draw[arrow] (chunking.south) -- ++(0,-0.8) -| (tts.north);
        \draw[arrow] (chunking.south) -- ++(0,-0.8) -| (ac.north);

        \draw[arrow] (mixer) -- (fade);
        \draw[arrow] (fade) -- (output);
    \end{tikzpicture}

    The text is robustly split using natural pauses (punctuation, newlines). For each chunk (mandated due to the strict \textbf{47-second generation maximum} of the under-the-hood Stable Audio model), the script runs the T5 text-to-speech engine concurrently with our Music Generation model. Finally, speech and music arrays are zero-padded to match lengths, multiplied by their specific gains, and blended with dynamic crossfades to avoid clipping artifacts.
}

\block{\MakeUppercase{Application III: Video Generation \& Orchestration}}{
    Taking the visual modality further, the \texttt{video\_sound.py} application creates animated cinematic clips with a tightly synchronized and responsive musical score from a single still image.
    \vspace{1cm}
    
    \textbf{Parallel Execution Pathway:}
    \vspace{0.5cm}
    \begin{enumerate}
        \item \textbf{Image Animation:} The input image is parsed through the \textbf{CogVideoX} Model. As we strictly utilized an \textit{open-weight model} for this task, the resulting video quality remains relatively limited but serves as a functional proof-of-concept for the pipeline.
        \item \textbf{Image Orchestration:} Simultaneously, the image goes directly into the \textbf{Audio\-Conditioner} model to extract visual semantics and produce a matching audio waveform.
        \item \textbf{Multimodal Merging:} We integrate \textbf{FFmpeg} through subprocesses to automatically compile the generated silent video (\texttt{.mp4}) and the generated audio (\texttt{.wav}), multiplexing them into a single presentation-ready file.
    \end{enumerate}
    
    \begin{center}

        node distance=2cm,

}

\block{\MakeUppercase{If We Had More Time}}{
    Given more time and compute budget, we would have fine-tuned our model using \textbf{Proximal Policy Optimization (PPO)}. By using the CLAP similarity score (between the generated audio and the text prompt) as a reward signal, the model could strictly optimize itself to directly maximize alignment instead of just distilling a Teacher LLM.
    \vspace{0.5cm}
    
    \begin{itemize}
        \item[\color{rouge485}\textbullet] \textbf{The Bottleneck:} In RL algorithms like PPO, the agent needs to continually generate outputs (rollouts) to explore the environment and receive rewards. 
        \item[\color{rouge485}\textbullet] \textbf{Order of Magnitude:} A single audio generation step with \textbf{Stable Audio} takes approximately \textbf{1 minute} to complete.
        \item[\color{bleu303}\textbullet] \textbf{Conclusion:} Gathering enough generation trajectories to meaningfully update the network via PPO would thus require \textbf{weeks of continuous computing time}, making it unfeasible within the scope of this project.
    \end{itemize}
}

\block{\MakeUppercase{References: Models Used}}{
    \begin{itemize}
        \item[\color{bleu303}\textbullet] \textbf{BLIP (Bootstrapping Language-Image Pre-training):} Visual scene semantic extraction and captioning.
        \item[\color{bleu303}\textbullet] \textbf{CLAP (Contrastive Language-Audio Pretraining):} Text semantic embeddings and cosine similarity candidate re-ranking.
        \item[\color{bleu303}\textbullet] \textbf{Stable Audio (Open):} Core diffusion model for structural 47-second music sequence generation.
        \item[\color{bleu303}\textbullet] \textbf{T5-TTS:} Robust text-to-speech synthesis engine utilized in the ReadMusic narrative pipeline.
        \item[\color{bleu303}\textbullet] \textbf{CogVideoX:} Advanced transformer-based text-to-video diffusion model for high-consistency visual animation.
    \end{itemize}
}
